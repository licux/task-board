# kanban-board

## Purpose

個人ローカル利用を前提とした、単一ボード / 3 列固定 (Todo / Doing / Done) のカンバン方式タスク管理 UI を提供する。カードはタイトルのみを持ち、追加・削除・ドラッグ & ドロップによる移動を通じてユーザーがタスクの進行状態を管理する。

## Requirements

### Requirement: 単一ボードの提供

システムは、ユーザーがアクセスした際に、単一のカンバンボードを表示 SHALL する。ボードの切り替えや複数ボード管理は提供しない。

#### Scenario: 初回アクセス時に単一ボードが表示される

- **WHEN** ユーザーがブラウザで `localhost` のトップページにアクセスする
- **THEN** システムは単一のカンバンボードを 1 画面で表示する
- **AND** 他のボードへ切り替える UI 要素は表示されない

---

### Requirement: 3 列固定カラム

ボードは `Todo`, `Doing`, `Done` の 3 つのカラムを、この左から右の順で固定的に MUST 表示する。カラムの追加・削除・並び替え・リネームは提供しない。

#### Scenario: 3 列が固定順で表示される

- **WHEN** ボードが表示される
- **THEN** カラムは左から `Todo`, `Doing`, `Done` の順で 3 つ表示される
- **AND** カラムを追加・削除・並び替え・リネームする UI 要素は表示されない

#### Scenario: カラム内にカードが無い場合の表示

- **WHEN** あるカラム内のカードが 0 枚である
- **THEN** そのカラムは見出しのみを表示し、カードエリアは空の状態で描画される
- **AND** ドロップ先としては引き続き機能する

---

### Requirement: カードの追加

ユーザーは `Todo` カラムに、タイトルを指定して新しいカードを追加できる SHALL。カードはタイトルのみを持ち、その他の属性は持たない。

#### Scenario: タイトルを入力して追加する

- **WHEN** ユーザーがタイトル入力欄にテキストを入力し、追加操作を実行する
- **THEN** システムは入力されたタイトルを持つ新しいカードを `Todo` カラムの末尾に追加する
- **AND** 追加されたカードは即座に画面に反映される
- **AND** 追加内容は永続化される (`task-persistence` capability に準拠)

#### Scenario: 空タイトルでの追加は拒否される

- **WHEN** ユーザーが空文字またはホワイトスペースのみのタイトルで追加操作を実行する
- **THEN** システムはカードを追加しない
- **AND** ユーザーが認識できる形で追加が行われなかったことが示される

---

### Requirement: カードの削除

ユーザーは、任意のカードを削除できる SHALL。削除は取り消しできない。

#### Scenario: カードを削除する

- **WHEN** ユーザーがあるカードに対して削除操作を実行する
- **THEN** システムは対象カードをボードから除去する
- **AND** 除去は永続化される (`task-persistence` capability に準拠)

#### Scenario: 削除は即時反映される

- **WHEN** カードの削除が実行される
- **THEN** 画面上のそのカードは即座に消える
- **AND** 復元用の UI (Undo) は提供されない

---

### Requirement: カードのドラッグ & ドロップによる移動

ユーザーは、任意のカードをドラッグ & ドロップにより、任意のカラム間で移動できる SHALL。同一カラム内での並び順変更は本 MVP のスコープには含まれるが、それ以外の並び替え (ボード間・列外への移動等) は提供しない。

#### Scenario: 別カラムへドロップする

- **WHEN** ユーザーがあるカードを別カラムへドラッグしてドロップする
- **THEN** システムはそのカードを移動先カラムの、ドロップされた位置に配置する
- **AND** 移動内容は永続化される (`task-persistence` capability に準拠)

#### Scenario: 同一カラム内で並び替える

- **WHEN** ユーザーがカードを同一カラム内の別の位置へドラッグしてドロップする
- **THEN** システムはそのカードを新しい位置に配置し、他カードの相対順序を維持する
- **AND** 並び替え結果は永続化される

#### Scenario: ドロップ先が無効な場合はキャンセルされる

- **WHEN** ユーザーがカードをカラム外の領域にドロップする、またはドラッグをキャンセルする
- **THEN** システムはカードの位置を変更しない
- **AND** ボードの状態は変わらない

---

### Requirement: フォールバック操作 UI を持たない

システムは、カラム間移動のためのドラッグ & ドロップ以外の操作手段 (移動ボタン、ドロップダウン等) を提供 MUST NOT。DnD が動作しない環境は本 MVP のサポート対象外である。

#### Scenario: ボタン等による移動 UI が存在しない

- **WHEN** ボードが表示されている
- **THEN** 各カードには「Doing に移動」「Done に移動」などのボタンやメニューは表示されない
- **AND** カラム移動はドラッグ & ドロップ操作でのみ可能である

---

### Requirement: 変更の即時可視化

システムは、カードの追加・削除・移動を実行した際に、画面上のボード状態を即時に更新 SHALL する。ページ全体のリロードを要求しない。

#### Scenario: 変更後にリロード不要で画面が更新される

- **WHEN** ユーザーが追加・削除・移動のいずれかを実行する
- **THEN** 変更結果は追加のページリロードなく画面に反映される
